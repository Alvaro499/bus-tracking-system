## Project Structure & Standards

### Backend (`/backend`)

- **Language**: Java 21

## Docker & Compose

### File Structure
- `docker-compose.yml` - Development only (PostgreSQL dev + PGAdmin)
- `docker-compose.test.yml` - Testing only (PostgreSQL test + PGAdmin)

### Quick Start (Development)
```bash
# Create environment files
cp .env.example .env          # Edit with your values
cp .env.test.example .env.test # Edit with your values

# Start development services
docker-compose up -d

# Verify
docker-compose ps
```

### Testing Setup
```bash
# Start testing services (separate from development)
docker-compose -f docker-compose.test.yml up -d

# Verify test containers
docker-compose -f docker-compose.test.yml ps

# Reset test database (clean for each run)
docker-compose -f docker-compose.test.yml down -v
docker-compose -f docker-compose.test.yml up -d
```

### Services Details

**Development (docker-compose.yml):**
- `postgres-dev` - PostgreSQL development BD (port 5432)
- `pgadmin` - Database UI (port 5050)

**Testing (docker-compose.test.yml):**
- `postgres-test` - PostgreSQL testing BD (port 5433)
- `pgadmin-test` - Database UI for testing (port 5051)

### Data Persistence
- Development BD: Volume `postgres_dev_data`
- Testing BD: Volume `postgres_test_data`
- Both: PGAdmin configs in separate volumes


### Never Commit
- `.env` files with credentials

## Code Style & Standards

### Java
- Java 21+
- Use Lombok for boilerplate (`@Data`, `@Slf4j`)
- Use records for immutable data classes
- Documentation: JavaDoc for public APIs

### SQL
- Always use prepared statements (prevent SQL injection)
- Comments for complex queries
- Migrations in `backend/src/main/resources/db/`

### Git Messages
- **Format**: `feat:`, `fix:`, `test:`, `docs:`, `chore:`
- **Example**: `feat: add user authentication endpoint`
- **Example**: `test: add unit tests for BusService`

## Common Tasks

### Start Development
```bash
docker-compose up -d
cd backend
mvn clean install
mvn spring-boot:run
```

### Run Tests
```bash
# Start test services first
docker-compose -f docker-compose.test.yml up -d

# Run all tests
mvn test

# Run specific test
mvn test -Dtest=BusServiceTest

# Run with coverage report
mvn test jacoco:report

# Stop test services after running
docker-compose -f docker-compose.test.yml down
```

### Access Database
- **PGAdmin**: http://localhost:5050
- **DBeaver**: Connect to `localhost:5432` (dev) or `localhost:5433` (test)

### Reset Everything
```bash
docker-compose down -v  # Remove volumes
docker-compose up -d    # Recreate from scratch
```
